\begin{abstract}

This report presents a comprehensive study of semantic video understanding through an integrated approach combining object detection and semantic embeddings. We analyze a state-of-the-art system that leverages YOLOv8 for real-time object detection and CLIP (Contrastive Language-Image Pre-training) for semantic understanding. The system implements a hybrid two-tier detection strategy with a primary fast path utilizing YOLO followed by semantic verification via CLIP embeddings, and a fallback mechanism employing sliding-window CLIP scans for detecting objects missed by YOLO. We investigate the architectural components, algorithmic foundations, temporal reasoning mechanisms, and practical implementation details. The analysis encompasses computational complexity, performance benchmarks, and practical applications in video search, safety monitoring, and compliance verification. Our findings demonstrate the effectiveness of combining classical object detection with modern semantic embeddings for robust and interpretable video analysis.

\end{abstract}

\textbf{Keywords:} Object Detection, CLIP Embeddings, Video Analysis, Semantic Understanding, YOLOv8, Temporal Reasoning

The analysis and understanding of video content remains a challenging problem in computer vision, particularly when semantic-level queries are required. Traditional approaches rely on object detection (e.g., YOLO) which can be fast but lacks semantic context, while purely semantic approaches (e.g., CLIP) can be computationally expensive when applied exhaustively to video streams. \begin{enumerate}
    \item Achieve real-time or near-real-time performance on video streams
    \item Provide semantic understanding beyond object class labels
    \item Robustly detect objects even when traditional detectors fail

    \item Maintain temporal coherence and reasoning about object persistence
\end{enumerate}

\section{Related Work}

\subsection{Object Detection}

Object detection has evolved significantly from traditional methods (e.g., Faster R-CNN, SSD) to real-time approaches. YOLOv8 (You Only Look Once version 8) represents the state-of-the-art in single-stage object detection, achieving excellent speed-accuracy tradeoffs through:

\begin{itemize}
    \item End-to-end differentiable architecture
    \item Multi-scale feature extraction
    \item Real-time inference on consumer-grade GPUs
\end{itemize}

The YOLO family \cite{yolo2016} has proven particularly effective for video analysis due to its computational efficiency.

\subsection{Vision-Language Models}

CLIP (Contrastive Language-Image Pre-training) \cite{clip2021} represents a significant advancement in connecting visual and semantic information. CLIP's architecture:

\begin{itemize}
    \item Learns joint embeddings of images and text
    \item Enables zero-shot classification without task-specific training
    \item Provides semantic understanding beyond predefined object classes
\end{itemize}

The application of CLIP to video analysis has opened new possibilities for semantic querying and understanding.

\subsection{Hybrid Approaches}

Recent work has explored combining classical detection with semantic embeddings. The rationale includes:

\begin{itemize}
    \item Leveraging speed advantages of detection-based approaches
    \item Augmenting with semantic understanding for robustness
    \item Implementing fallback mechanisms for improved recall
\end{itemize}

\section{System Architecture}
% ============================================================================

\subsection{High-Level Overview}

The system implements a five-stage pipeline for video analysis:

\begin{enumerate}
    \item \textbf{Model Loading}: Initialization of YOLOv8 and CLIP models
    \item \textbf{Video Processing}: Frame extraction and sampling
    \item \textbf{Object Detection}: Multi-modal detection strategy
    \item \textbf{Query Matching}: Natural language query resolution
    \item \textbf{Post-Processing}: Temporal aggregation and reasoning
\end{enumerate}

\subsection{Hybrid Detection Strategy}

The core innovation is a two-tier detection approach:

\textbf{Tier 1 (Fast Path):} Rapid object detection followed by semantic verification
\begin{itemize}
    \item Apply YOLOv8 model to each frame
    \item Extract object crops with contextual padding (25\%)
    \item Compute CLIP embeddings for crops
    \item Match against query embedding using cosine similarity
    \item Apply label-based semantic boost
\end{itemize}

\textbf{Tier 2 (Fallback Path):} Comprehensive sliding-window search when primary detection fails
\begin{itemize}
    \item Apply multi-scale sliding window over frame
    \item Evaluate CLIP similarity at each window
    \item Use zero-shot classification with negative prompts
    \item Apply non-maximum suppression
    \item Return candidates exceeding confidence threshold
\end{itemize}

The decision logic:

\begin{equation}
\text{detections} = \begin{cases}
T1(frame) & \text{if } |T1(frame)| > 0 \text{ and match found} \\
T2(frame) & \text{otherwise}
\end{cases}
\end{equation}

where $T1$ and $T2$ represent Tier 1 and Tier 2 respectively.

\subsubsection{Model Initialization Module}

\textbf{CLIP Model Wrapper:} Provides unified interface for text and image encoding

\begin{lstlisting}[caption=CLIP Model Initialization]
def load_clip(model_name="ViT-B/32"):
    """Load CLIP model with device selection"""
    device = "cuda" if torch.cuda.is_available() \
             else "cpu"
    model, preprocess = clip.load(model_name, 
                                  device=device)
    return model, preprocess, device

def encode_text(text):
    """Encode text to normalized embedding"""
    model, _, device = load_clip()
    tokens = clip.tokenize([text]).to(device)
    with torch.no_grad():
        embedding = model.encode_text(tokens)
    return embedding / embedding.norm(dim=-1, 
                                     keepdim=True)

def encode_image(bgr_crop):
    """Encode image to normalized embedding"""
    import cv2
    model, preprocess, device = load_clip()
    rgb = cv2.cvtColor(bgr_crop, 
                      cv2.COLOR_BGR2RGB)
    pil_img = Image.fromarray(rgb)
    img_tensor = preprocess(pil_img).unsqueeze(0)
    img_tensor = img_tensor.to(device)
    with torch.no_grad():
        embedding = model.encode_image(img_tensor)
    return embedding / embedding.norm(dim=-1, 
                                     keepdim=True)
\end{lstlisting}

Key design decisions:

\begin{itemize}
    \item Lazy initialization: Models loaded on first use
    \item L2-normalization: Enables cosine similarity computation
    \item Device abstraction: Automatic GPU/CPU selection
    \item Caching: Global state prevents redundant loads
\end{itemize}

\subsubsection{Tier 1: YOLO-Based Detection}

\begin{lstlisting}[caption=Primary Detection Pipeline]
def detect_objects(model, frame, 
                   conf_threshold=0.25):
    """YOLO inference with context extraction"""
    results = model(frame, verbose=False)[0]
    detections = []
    h, w = frame.shape[:2]
    
    for box in results.boxes:
        conf = float(box.conf[0])
        if conf < conf_threshold:
            continue
        
        x1, y1, x2, y2 = map(int, 
                    box.xyxy[0].tolist())
        label = results.names[int(box.cls[0])]
        
        # Tight crop for color analysis
        tight_crop = frame[y1:y2, x1:x2]
        color = get_dominant_color(tight_crop)
        
        # Padded crop (25 percent) for CLIP
        bw, bh = x2 - x1, y2 - y1
        pad_x = int(bw * 0.25)
        pad_y = int(bh * 0.25)
        cx1 = max(0, x1 - pad_x)
        cy1 = max(0, y1 - pad_y)
        cx2 = min(w, x2 + pad_x)
        cy2 = min(h, y2 + pad_y)
        clip_crop = frame[cy1:cy2, cx1:cx2]
        
        detections.append({
            "bbox": [x1, y1, x2, y2],
            "label": label,
            "confidence": conf,
            "color": color,
            "crop": clip_crop,
        })
    
    return detections
\end{lstlisting}

Analysis of design choices:

\begin{enumerate}
    \item \textbf{Padding Strategy}: 25\% contextual padding provides sufficient surrounding region for CLIP without excessive computational overhead
    \item \textbf{Dual Cropping}: Tight crop for color analysis (object-centric), padded crop for semantic analysis (contextual)
    \item \textbf{Early Termination}: Confidence thresholding reduces downstream processing
\end{enumerate}

\subsubsection{Tier 2: CLIP Sliding Window Scan}

\begin{lstlisting}[caption=Fallback Detection with Zero-Shot Filtering, 
                  firstline=1, lastline=40]
def clip_scan(frame, query, threshold=0.25, 
              grid_size=(6, 8), 
              scales=(0.15, 0.10, 0.08)):
    """Multi-scale sliding window with verification"""
    query_text = "a photo of a {}".format(query)
    negatives = [
        "a photo of a plain background",
        "a photo of a wall",
        "a photo of a floor",
        "a photo of a shadow",
        "a photo of furniture"
    ]
    all_texts = [query_text] + negatives
    
    # Encode all prompts
    text_embs = torch.stack([
        encode_text(t).squeeze() 
        for t in all_texts
    ])
    query_emb = text_embs[0].unsqueeze(0)
    
    candidates = []
    h, w = frame.shape[:2]
    
    for scale in scales:
        win_h = int(h * scale)
        win_w = int(w * scale)
        if win_h < 20 or win_w < 20:
            continue
        
        rows, cols = grid_size
        step_y = max(1, (h - win_h) // rows)
        step_x = max(1, (w - win_w) // cols)
        
        for y in range(0, h - win_h + 1, step_y):
            for x in range(0, w - win_w + 1, 
                          step_x):
                crop = frame[y:y + win_h, 
                           x:x + win_w]
                img_emb = encode_image(crop)
\end{lstlisting}

Zero-shot filtering mechanism:

\begin{equation}
P(\text{query} | \text{crop}) = \frac{\exp(s_{\text{query}} / \tau)}{\sum_{i=0}^{n} \exp(s_i / \tau)}
\end{equation}

where $s_i = \cos(\text{text}_i, \text{image})$ and $\tau = 1.0$ (temperature).

Acceptance criterion:

\begin{equation}
\text{accept} = P(\text{query} | \text{crop}) > 0.40
\end{equation}

This threshold balances recall (detecting objects) with precision (avoiding false positives).

\subsubsection{Hybrid Matching Module}

\begin{lstlisting}[caption=Query-Object Matching with Label Boost]
def match_crops(query, detections, 
                threshold=0.20):
    """Match text query against detections"""
    query_emb = encode_text(query)
    matches = []
    
    for det in detections:
        crop = det["crop"]
        if crop.size == 0:
            continue
        
        # CLIP similarity
        img_emb = encode_image(crop)
        sim_base = torch.nn.functional \
            .cosine_similarity(
                query_emb, img_emb)
        sim_base = float(sim_base)
        
        # Apply label boost
        sim_final = _label_boost(query, 
                          det.get("label", ""), 
                          sim_base)
        
        if sim_final >= threshold:
            det_copy = {k: v for k, v in 
                       det.items() 
                       if k != "crop"}
            det_copy["similarity"] = sim_final
            matches.append(det_copy)
    
    return matches

def _label_boost(query, label, base_sim):
    """Semantic label enhancement"""
    query_words = set(query.lower().split())
    label_lower = label.lower()
    
    # Direct match: strong boost
    if label_lower in query_words:
        return base_sim + 0.08
    
    # Partial match: weak boost
    for word in query_words:
        if word in label_lower or \
           label_lower in word:
            return base_sim + 0.05
    
    return base_sim
\end{lstlisting}

Mathematical formulation of label boost:

\begin{equation}
\text{sim}_{\text{final}} = \text{sim}_{\text{CLIP}} + \text{boost}(\text{query}, \text{label})
\end{equation}

\begin{equation}
\text{boost}(\text{query}, \text{label}) = \begin{cases}
0.08 & \text{if } \text{label} \in \text{query words} \\
0.05 & \text{if } \text{label} \text{ partially matches} \\
0.00 & \text{otherwise}
\end{cases}
\end{equation}

Rationale: YOLO label information provides complementary signal to CLIP embeddings, particularly useful for distinguishing between similar objects (e.g., different types of vehicles).

\subsubsection{Frame Streaming}

\begin{lstlisting}[caption=Adaptive Frame Sampling]
def stream_frames(video_path, sample_fps=1):
    """Sample frames at configurable rate"""
    cap = cv2.VideoCapture(video_path)
    if not cap.isOpened():
        raise ValueError(
            "Cannot open video: {}".format(
                video_path))
    
    video_fps = cap.get(cv2.CAP_PROP_FPS)
    if video_fps <= 0:
        raise ValueError(
            "Could not determine video FPS")
    
    frame_interval = int(video_fps / sample_fps)
    frame_idx = 0
    
    while True:
        ret, frame = cap.read()
        if not ret:
            break
        
        if frame_idx \% frame_interval == 0:
            timestamp = frame_idx / video_fps
            yield frame, timestamp
        
        frame_idx += 1
    
    cap.release()
\end{lstlisting}

Design considerations:

\begin{itemize}
    \item Adaptive sampling maintains temporal accuracy
    \item Configurable FPS enables speed-accuracy tradeoff
    \item Memory-efficient streaming with generator pattern
\end{itemize}

\subsubsection{Temporal Tracking}

\begin{lstlisting}[caption=Detection History Management]
class DetectionTracker:
    """Maintains temporal detection history"""
    
    def __init__(self):
        self.history = []
    
    def add(self, timestamp, detections):
        """Record detections at timestamp"""
        self.history.append((timestamp, 
                            detections))
    
    def get_all(self):
        """Retrieve full history"""
        return self.history
    
    def get_timestamps_with_label(self, label):
        """Query timestamps by object label"""
        timestamps = []
        for ts, dets in self.history:
            for d in dets:
                if d.get("label") == label:
                    timestamps.append(ts)
                    break
        return timestamps
\end{lstlisting}

\subsubsection{Temporal Aggregation}

\begin{lstlisting}[caption=Segment Grouping Algorithm]
def aggregate_segments(timestamps, 
                      gap_threshold=2.0):
    """Group timestamps into coherent segments"""
    if not timestamps:
        return []
    
    timestamps = sorted(timestamps)
    segments = []
    seg_start = timestamps[0]
    seg_end = timestamps[0]
    
    for t in timestamps[1:]:
        if t - seg_end <= gap_threshold:
            seg_end = t
        else:
            segments.append((seg_start, 
                           seg_end))
            seg_start = t
            seg_end = t
    
    segments.append((seg_start, seg_end))
    return segments
\end{lstlisting}

Algorithm explanation:

\textbf{Input:} Sorted list of detection timestamps

\textbf{Process:}
\begin{enumerate}
    \item Initialize segment with first timestamp
    \item Iterate through remaining timestamps
    \item If gap $ \leq $ threshold, extend current segment
    \item Otherwise, finalize segment and start new one
    \item Finalize last segment
\end{enumerate}

\textbf{Complexity:} $O(m \log m)$ dominated by sorting, where $m = $ number of timestamps.

\textbf{Rationale:} Default threshold of 2.0 seconds balances:
\begin{itemize}
    \item Grouping brief interruptions in detection
    \item Separating distinct object occurrences
\end{itemize}

\subsection{Color Classification}

\begin{lstlisting}[caption=HSV-Based Color Detection]
def get_dominant_color(bgr_crop):
    """Extract dominant color via HSV analysis"""
    if bgr_crop.size == 0:
        return "unknown"
    
    h, w = bgr_crop.shape[:2]
    if h < 4 or w < 4:
        return "unknown"
    
    # Extract center region
    cy1, cy2 = h // 4, 3 * h // 4
    cx1, cx2 = w // 4, 3 * w // 4
    center = bgr_crop[cy1:cy2, cx1:cx2]
    
    # Normalize and convert to HSV
    center = cv2.resize(center, (24, 24))
    hsv = cv2.cvtColor(center, 
                      cv2.COLOR_BGR2HSV)
    
    # Robust statistics using median
    med_h = float(np.median(hsv[:, :, 0]))
    med_s = float(np.median(hsv[:, :, 1]))
    med_v = float(np.median(hsv[:, :, 2]))
    
    return _hsv_to_color_name(med_h, med_s, 
                             med_v)

def _hsv_to_color_name(h, s, v):
    """HSV to color name mapping"""
    # Achromatic classification
    if s < 40:
        if v < 60:
            return "black"
        elif v < 160:
            return "gray"
        else:
            return "white"
    
    # Chromatic classification by hue
    if h <= 8 or h >= 165:
        return "red"
    elif 9 <= h <= 20:
        return "orange"
    elif 21 <= h <= 35:
        return "yellow"
    elif 36 <= h <= 85:
        return "green"
    elif 86 <= h <= 100:
        return "cyan"
    elif 101 <= h <= 130:
        return "blue"
    elif 131 <= h <= 145:
        return "purple"
    elif 146 <= h <= 164:
        return "pink"
    
    return "unknown"
\end{lstlisting}

Color classification design:

\begin{enumerate}
    \item \textbf{Center-Focused}: Ignores background by analyzing center 50\%
    \item \textbf{Robust Statistics}: Median instead of mean for outlier resistance
    \item \textbf{Two-Stage}: First classify chromatic vs. achromatic, then apply hue ranges
\end{enumerate}

\subsubsection{Sliding Window Analysis}

\begin{equation}
T_{\text{window}} = O(s \cdot g^2 \cdot c)
\end{equation}

where:
\begin{itemize}
    \item $s$ = number of scales (typically 3)
    \item $g$ = grid dimension (typically 6-8)
    \item $c$ = cost of CLIP encoding + similarity computation
\end{itemize}

Total complexity for sliding window:

\begin{equation}
T_{\text{total}} = s \sum_{i=1}^{s} g_i^2 \cdot c = O(s^2 g^2 c)
\end{equation}

For default parameters: $T \approx 3 \cdot 6^2 \cdot \text{CLIP\_cost} = 108 \cdot \text{CLIP\_cost}$

\section{Limitations and Future Work}
% ============================================================================

\subsection{Current Limitations}

\begin{enumerate}
    \item \textbf{Single Object Tracking}: No multi-object correspondence
    \item \textbf{Static Rules}: Compliance rules are hand-crafted
    \item \textbf{Computational Cost}: Fallback mechanism is expensive
    \item \textbf{Color Dependence}: Color classification may fail for extreme lighting
    \item \textbf{No Temporal Patterns}: Limited reasoning about object dynamics
\end{enumerate}

\subsection{Future Enhancements}

\begin{enumerate}
    \item \textbf{Multi-Object Tracking}: Implement Hungarian algorithm or Kalman filtering for trajectory matching
    \item \textbf{Learned Compliance}: Use neural networks to learn violation patterns
    \item \textbf{Optimized Scanning}: Implement spatial-temporal optimization for Tier 2
    \item \textbf{Action Recognition}: Extend to detect object interactions and events
    \item \textbf{Real-Time Streaming}: Support live video feed processing
\end{enumerate}

\section{Conclusion}
% ============================================================================

This work presents a comprehensive analysis of semantic video understanding through hybrid detection combining classical object detection with modern semantic embeddings. The two-tier approach balances speed and robustness, providing practical utility for real-world applications.

\textbf{Key findings:}

\begin{enumerate}
    \item Hybrid detection strategies effectively combine YOLO's speed with CLIP's semantic understanding
    \item Fallback mechanisms improve robustness at reasonable computational cost
    \item Temporal aggregation enables meaningful event summarization
    \item Practical applications (safety compliance) are feasible and valuable
\end{enumerate}

\textbf{Significance:}

\begin{itemize}
    \item Demonstrates viable approach to semantic video analysis
    \item Provides architectural template for similar applications
    \item Highlights importance of hybrid approaches in computer vision
\end{itemize}

The system successfully addresses the challenge of combining real-time performance with semantic understanding, relevant to industrial and surveillance applications requiring both efficiency and interpretability.
